# 品目コード（統計品目番号）と国名符号

## 統計品目番号（9桁HSコード）

貿易統計の品目は **9桁の統計番号** で集計される。構成:

```
  09 01 11 . 000
  └┬┘ └──┬──┘ └┬┘
  類   国際HS6   日本独自の細分(9桁化)
 (章)
```

- 先頭2桁 = 類（章, HS Chapter）。例: `09`=コーヒー・茶・香辛料, `18`=ココア及びその調製品, `27`=鉱物性燃料（原油等）。
- 6桁 = 国際的に共通の HS コード。
- 9桁 = 日本の統計細分。

**正確な9桁は必ず確認する**:
- API のフィルタに使うコードは `getMetaInfo`（`estat.ts meta --id ...`）が返す `cat01` のコードが正。
- 人間向けの一覧は税関の「輸出統計品目表」「輸入統計品目表（実行関税率表）」。

### よく追跡する品目の章/項（参考）

| 品目 | 章(2桁) | 主な項(HS4) | 備考 |
|---|---|---|---|
| コーヒー | 09 | 0901 | 生豆/焙煎/カフェインレス等で9桁細分 |
| ココア（豆・調製品） | 18 | 1801（豆）, 1806（チョコ等） | |
| 原油・石油製品 | 27 | 2709（原油）, 2710（石油製品） | |

> 章レベル（2桁）で大づかみに見るなら `lvCat01` で階層を絞るか、`cdCat01From/To` で範囲指定。
> 特定9桁を狙うなら `meta` でコードを確定してから `cdCat01` に渡す。

## 国名符号（国・地域コード）

税関の「統計国名符号表」に基づく国・地域コードが使われる。
API での実コードは表により `area` 次元または `cat02` 次元に入る — **`meta` で paramId と実コードを確認**してから `cdArea`（または `cdCat02`）に渡す。

- 主要な集計地域: 全世界, アジア, ＡＳＥＡＮ, ＥＵ, 中東 などの地域合計コードも存在する。
- 概況品検索や報道発表資料では「全世界 / アメリカ合衆国 / ＥＵ / アジア / 中華人民共和国 / ＡＳＥＡＮ / 中東 / ロシア」の主要区分が用意されている。

## 実務上の鉄則

1. コードを推測で書かない。`estat.ts meta --id <statsDataId> --full` で次元と全コードをダンプし、目的の品目・国の正確なコードを得る。
2. 名称でのあたりを付けるなら、`analyze.ts` の `--cat` / `--area` は **コードでも名称（部分一致）でも** マッチするので、まず全件 `fetch` してから名称でフィルタする探索的な使い方も有効。
3. 単位は次元 `tab` と各品目で異なる（金額=千円、数量=kg/個/L等）。`tab` を金額に固定してから国別比較・時系列にすると整合的。
