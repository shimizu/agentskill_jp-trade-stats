# 財務省貿易統計 — 統計表カタログとデータ構造

財務省貿易統計（Trade Statistics of Japan）は e-Stat 経由で機械可読形式で提供される。
税関サイト本体（customs.go.jp）は robots.txt で自動取得を拒否しているため、**プログラムからのアクセスは必ず e-Stat API を使う**。

## 政府統計コード（statsCode）

| 統計 | statsCode |
|---|---|
| 普通貿易統計（一般に「貿易統計」） | `00350300` |

`getStatsList?statsCode=00350300` で配下の全統計表（statsDataId）を列挙できる。これがプログラム的なテーブル発見の起点。

## 統計の3区分

1. **普通貿易統計** — 輸出入貨物の金額・数量を品目別・国（地域）別等に集計。最も一般的。本スキルの主対象。
2. **特殊貿易統計** — 金統計、船用品・機用品統計、通過貿易統計。
3. **船舶・航空機統計** — 国籍別の船舶・航空機の入出港統計。

## 普通貿易統計の主要な統計表タイプ

e-Stat の階層（`貿易統計_全国分`）配下に以下がある。各々に輸出/輸入・月次がある。

| 表名 | 内容 | 品目の粒度 |
|---|---|---|
| **品別国別表** | 統計品目（9桁HS）× 国 × 月 の金額・数量 | 9桁統計品目番号（最も細かい） |
| **国別概況品別表** | 国 × 概況品 の金額・数量 | 概況品（品目をまとめた中分類） |
| **概況品別統計品目表** | 概況品 × 配下の統計品目 | 概況品＋9桁 |
| **貿易指数表** | 数量指数・価格指数・金額指数 | 指数 |
| 統計品別表（全国分） | 品目別の全国計 | 9桁 |

### 速報・確報・確定の区別（重要）

同じ表でも公表段階が複数ある。新しい順に確定 > 確々報 > 確報 > 速報。
- **速報**: 月末直後。輸入は9桁速報あり。暫定値。
- **確報 / 確々報 / 確定**: 後から確定値に更新。分析・時系列には確報以降を使う。

`getStatsList` の結果タイトルに「速報」「確報」「確定」が含まれるので、用途に応じて選ぶ。年計が欲しい場合は12月分の累計欄を見る（または年次表を使う）。

## データの次元（dimension）構造 — 必ず getMetaInfo で確認する

表ごとに次元の API パラメータ ID（`tab` / `cat01` / `cat02` / `area` / `time`）への割り当てが異なる。
**ハードコードせず、`estat.ts meta --id <statsDataId>` で実際の割り当てを毎回確認すること。**

実測した割り当て（品別国別表 輸入 `0004049326` / `0003425294` 等）:

| paramId | 次元名 | 例 | API フィルタキー |
|---|---|---|---|
| `cat01` | 統計品目（9桁HS）または概況品 | `090111000`（コーヒー生豆）, `270900`（原油）等 | `cdCat01`, `cdCat01From/To` |
| `cat02` | **数量・金額の別 かつ 月の別** | `100`=単位1, `120/130`=合計_数量1/2, **`140`=合計_金額**, **`170`=1月_金額**, `200`=2月_金額 … | `cdCat02`, `cdCat02From/To` |
| `area` | 国・地域 | `50410`（ブラジル）, アメリカ合衆国 等 | `cdArea`, `cdAreaFrom/To` |
| `time` | 時間軸 | **年次のみ**（`2026000000` = 2026年, `2021000000` = 2021年） | `cdTime`, `cdTimeFrom/To` |

- **重要: 品別国別表には独立した `tab`（表章項目）次元が無い。** 金額/数量の区別も「○月／合計」の区別も
  すべて `cat02` に格納される。比較・集計の前に、`cat02` を目的の計測に必ず固定すること
  （年計金額なら `--cdCat02 140`、1月の金額なら `--cdCat02 170` …）。固定しないと数量・金額・12か月分が
  混在して合計が無意味になる。`analyze.ts` の `--tab` 名でこの表を絞ることはできない（tab 次元が無いため）。
- 金額の単位は通常 **千円（1,000 YEN）**。数量は品目ごとに単位が異なる（kg, 個, L 等）。`meta` の各項目の `unit` を確認。
- **時間軸コードは年次のみ**（`yyyy000000`）で、月の情報は time に入らない（月は `cat02`）。
  したがって `analyze.ts` の yoy は「同じ `cat02`（＝同じ計測・同じ月 or 年計）を固定したうえで、複数年の
  年次表を跨いで年次同士を比較する」動きになる。前年同月比が欲しければ `--cdCat02` をその月の金額コード
  （例 `170`=1月）に固定し、複数年を含む年次表に対して yoy を実行する。
- 上記は実測値だが、表タイプ・輸出/輸入で割り当てが変わりうる。**必ず `estat.ts meta --id <id>` で毎回確認すること。**

## statsDataId の調べ方（推奨フロー）

1. `estat.ts list --word "品別国別表 輸出 確報"` のようにキーワード検索、または `--word` 無しで `statsCode=00350300` 配下を一覧。
2. タイトルの「輸出/輸入」「速報/確報/確定」「対象年」で目的の表を選び、`statsDataId` を取得。
3. `estat.ts meta --id <statsDataId>` で次元と利用可能コードを確認。
4. `estat.ts fetch --id <statsDataId> --cd... ` でデータ取得。

## 既知の statsDataId 例（参考・変動しうる）

公表のたびに新しい statsDataId が発行される（過去年を含む通期表が更新される）。固定値に依存せず毎回 `list` で確認するのが安全。例として確認された値:

- `0003425293` … 品別国別表 輸出 確報（2021–2024確定, 2025確報）

> 注意: 上記は一例。時間が経つと別IDに置き換わるため、必ず `list` で最新を取得すること。
