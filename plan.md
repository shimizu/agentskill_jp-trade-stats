# 単位計算まわりの改善計画

## 背景

取得済みデータ `out/data/*.json` を確認した結果、現在の集計対象はすべて金額データで、単位は `千円` だった。既存ビルダーの換算式も妥当。

- 千円 → 億円: `/ 100_000`
- 千円 → 兆円: `/ 1_000_000_000`

そのため、直ちに修正が必要な計算ミスは見つかっていない。

## 提案 1: ビルダー入力の単位検証を追加する

### 対象

- `template/.agent/skills/jp-trade-stats/scripts/builders/chapter-aggregate-annual-report.ts`
- `template/.agent/skills/jp-trade-stats/scripts/builders/chapter-aggregate-monthly-series.ts`

### 内容

入力JSONを読むときに、想定外の単位・計測コードが混ざっていないか検証する。

年次ビルダー:

- `unit === "千円"` であること。
- `cat02 === "140"` であること。

月次ビルダー:

- `unit === "千円"` であること。
- `cat02` が月別金額コード `170, 200, 230, 260, 290, 320, 350, 380, 410, 440, 470, 500` のいずれかであること。

違反があれば、集計を続けずエラーにする。

### 目的

数量データや別単位データを誤って合算する事故を防ぐ。

## 提案 2: 月次CSVの億円表示の丸めを明示する

### 対象

- `template/.agent/skills/jp-trade-stats/scripts/builders/chapter-aggregate-monthly-series.ts`

### 内容

現在の `合計_億円` は整数丸め。誤りではないが、列名か出力値で丸めを明確にする。

選択肢:

- 列名を `合計_億円_概算` にする。
- または小数1桁で出力する。

### 推奨

既存CSV利用者への影響を抑えるなら、値は整数のまま列名を `合計_億円_概算` に変更する。

## 見送ること

- すべての単位を自動換算する汎用単位変換機能。
- 数量単位、重量単位、金額単位を横断する複雑な正規化。
- `estat.ts fetch` 側で単位ごとに出力形式を変えること。

現時点では、ビルダー入力の検証だけで十分。

## 実施順

1. 年次ビルダーに `unit` と `cat02` の検証を追加する。
2. 月次ビルダーに `unit` と月別金額コードの検証を追加する。
3. 月次CSVの `合計_億円` 列名を `合計_億円_概算` に変更するか、小数1桁出力に変更する。
4. `npm run build` を実行する。
5. 既存 `out/data/*.json` でビルダーを再実行し、生成物が期待通り作られることを確認する。
